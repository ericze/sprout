# 初长 V1 上线拍板清单

**用途：** 用最少的产品决策，解除当前上线准备工作的阻塞项。

**当前状态：** 已根据 2026-04-05 的产品拍板结果填写。

---

## 已确认决策

- 范围：`仅 local-first`，公开 V1 中移除 `cloud`、`family`、`paywall`
- 语言：`双语并支持 App 内切换`
- 首发语言：`中文 / 英文`
- 成长页英文标题：`Growth`
- 英文辅食标签策略：`改成更窄、更偏食材描述的一组标签，并保留 First taste`
- 通知：`V1 移除通知权限请求`

---

## D01：首发版本范围

**最终决策**

- 最终采用：`方案 A`
- V1 明确排除：`CloudKit sync / family sharing / subscriptions / paywall`
- V1 允许展示的侧边栏 / 设置项：
  - 仅保留已完成且真实可用的入口
  - 移除 `cloud`
  - 移除 `family`
  - 移除公开路径中的 `paywall`

**工程落地要求**

- 从公开 V1 中移除或隐藏所有未闭环能力入口
- 侧边栏和设置页不能再出现“coming soon”式死路
- 工程实现必须以 `local-first + 免费 + 单宝宝` 为前提，不为本轮未上线能力保留显性 UX

**完成标准**

- 用户在 V1 中只能看到真实可用的功能入口
- 不存在通往 `cloud`、`family`、`paywall` 的公开路径

---

## D02：语言策略

**最终决策**

- 最终采用：`方案 B`
- 首发语言：`简体中文 + 英文`
- App 内语言切换：`保留，并且必须真实生效`
- 成长页英文标题：`Growth`
- 英文辅食标签策略：
  - 不直接照搬当前中文标签
  - 改成更窄、更偏食材描述的一组标签
  - 保留 `First taste`

**工程落地要求**

- App 内语言切换不能只是本地 UI 状态，必须真正驱动全局语言变化
- Treasure / Onboarding / Growth / Shell / 设置页中的硬编码中文都要清干净
- 默认名、迁移逻辑、日期/月份文案、周信内容都要满足双语发布要求
- 英文辅食标签需要重新定义一组收敛后的可本地化文案，而不是机械直译

**产品补充说明**

- 成长页英文标题使用 `Growth`
- 英文辅食标签不采用“逐字翻译当前中文标签”的策略
- `First taste` 需要保留在英文体系中

**完成标准**

- App 内切换中英文后，核心页面文案、格式化内容和设置文案都正确切换
- 不再依赖中文默认值推断用户状态
- 不存在明显的中英混杂或失效切换路径

---

## D03：通知策略

**最终决策**

- 最终采用：`方案 A`
- V1 不请求通知权限
- V1 不上线提醒能力

**工程落地要求**

- 从 onboarding 中移除通知权限请求
- 清理与通知相关但本版无实际用途的 entitlement / background 配置
- 设置页中不再出现与提醒能力相关的误导入口

**完成标准**

- 用户在 V1 使用流程中不会被请求通知权限
- 工程配置不再声明本版未实际交付的通知能力
