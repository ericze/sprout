下面继续，给出 《初长 V1.0｜珍藏模块异常 / 空态 / 边界清单》。 这份文档的目标是：把珍藏模块在真实使用中最容易被忽略、但最容易出问题的场景一次性列清，供 Codex、测试和联调直接参照执行。

初长 V1.0｜珍藏模块异常 / 空态 / 边界清单
文档用途：交给 Codex / iOS 开发 / 联调 / QA 模块范围：珍藏（照片、日记、里程碑、AI 周信） 目标：定义珍藏模块在空态、异常态、低内容态、交互边界和状态冲突下的处理规则

1. 总体原则
珍藏模块的异常处理必须遵守：
	1	不制造“页面坏了”的感觉
	2	不暴露底层技术细节
	3	优先保留内容，其次才是修饰
	4	宁可降级，也不要中断阅读
	5	所有边界行为都要保持珍藏页的安静气质

2. 页面级空态与低内容态

2.1 完全空态（0 内容）
触发条件
满足以下全部：
	•	无 MemoryEntry
	•	无 WeeklyLetter
	•	无任何珍藏内容
页面表现
	•	不显示空白时间流
	•	居中展示极简空态
UI 组成
	•	线稿相机图标（鼠尾草绿，30% 透明）
	•	文案：这里存放时间。点击右上角，留住今天。
允许
	•	右上角 + 留住今天 入口可点击
禁止
	•	不弹出系统教学弹窗
	•	不出现“快来创建第一条内容”式平台 CTA
	•	不自动打开记录页

2.2 低内容态（1–2 条内容）
触发条件
	•	有 1–2 条 MemoryEntry 或
	•	只有极少内容 + 可能存在周信
页面表现
	•	仍然使用单列大画幅时间流
	•	不切换成小图模式
	•	不缩小卡片
	•	不额外增加说明文案
原则
低内容不是异常，只是早期使用阶段。

2.3 某个过滤器结果为空
A. 星标时刻为空
触发条件
	•	当前过滤器为 starredMoments
	•	无任何 isMilestone = true 的内容
表现
显示轻空态：
	•	文案：还没有被点亮的时刻。
禁止
	•	不提示“去创建一个里程碑”
	•	不做任务式引导

B. 时光信笺为空
触发条件
	•	当前过滤器为 timeLetters
	•	尚未生成任何 WeeklyLetter
表现
显示轻空态：
	•	文案：这里会放时间寄来的信。
禁止
	•	不提示“继续记录以解锁周信”
	•	不把周信做成成就系统

3. 内容级异常态

3.1 图片路径失效
触发条件
MemoryEntry.imageLocalPath 存在，但本地文件不可读或加载失败
页面表现
	•	不能让整条卡片消失
	•	优先降级为文字卡
处理规则
有 note
	•	直接退化为纯文字记忆卡
	•	可选显示极轻文案：这张照片暂时没有加载出来。
无 note
	•	该条内容进入异常态
	•	不建议正常展示为“空白卡片”
原则
优先保留文字内容；若图文都不可用，则不应假装内容正常。

3.2 只有图片，无文字
触发条件
	•	imageLocalPath != nil
	•	note == nil 或为空
页面表现
	•	合法展示
	•	仍然使用标准图片卡
	•	时间戳照常显示
	•	文本区可收窄，但不能消失得像布局断裂
原则
用户只想留一张照片是合法行为。

3.3 只有文字，无图片
触发条件
	•	imageLocalPath == nil
	•	note 非空
页面表现
	•	展示纯文字卡
	•	保留时间戳
	•	不预留图片区空白
原则
珍藏模块不是照片强依赖型产品。

3.4 只有里程碑开关，无图无文
触发条件
	•	isMilestone == true
	•	imageLocalPath == nil
	•	note 为空
保存规则
	•	不允许保存
关闭行为
	•	视为“用户有过操作”
	•	若用户尝试关闭输入页，应弹出放弃确认
原则
“打星”不是内容本身。

4. 里程碑边界场景

4.1 第一行过长
触发条件
	•	第一行 > 18 个汉字
	•	或无清晰断句
处理
	•	不做轻标题强化
	•	全文保持 Regular
禁止
	•	不强行截断成标题
	•	不自动抽关键词
	•	不自动制造副标题

4.2 用户主动换行，但第一行非常短
触发条件
用户主动换行，例如：
	•	会走了
	•	换行
	•	后面一大段
处理
	•	允许第一行作为轻强化标题
	•	因为用户已明确表达结构意图
原则
主动换行优先于算法判断。

4.3 过滤到“星标时刻”时只有 1 条
表现
	•	正常展示单条里程碑卡
	•	不额外加说明
	•	不因为只有一条而改变布局

5. AI 周信边界场景

5.1 静默周完全无珍藏内容
触发条件
	•	无照片
	•	无日记
	•	无里程碑
	•	珍藏模块基本没被使用
结果
	•	仍可生成 silent 类型周信
	•	表现为极窄文本卡
	•	不生成信封
原则
静默周是“轻轻留白”，不是“什么都没有”。

5.2 普通周内容不足
触发条件
例如：
	•	只有 1 条极弱内容
	•	1 张无说明、无上下文的照片
	•	1 句非常短的无信息文本
处理
	•	不强行升级成普通周信
	•	应优先回退到 silent
原则
不是有一点内容就值得一封信。

5.3 高内容密度周没有里程碑，但有很多内容
触发条件
	•	无 isMilestone = true
	•	但照片 / 日记数量 >= 5 或
	•	有明显连续叙事内容
处理
	•	仍可生成 dense
	•	因为“高内容密度”不等于“有里程碑”
原则
密度来自内容厚度，不只来自星标。

5.4 周信展开后切换过滤器
触发条件
	•	当前周信处于展开态
	•	用户点击切换到其他过滤器
处理
	•	先关闭 / 销毁当前周信展开态
	•	再切换过滤器
	•	不允许展开态残留到新列表中
原则
过滤结果必须是干净重建的。

5.5 周信内容生成失败
触发条件
	•	AI 生成失败 或
	•	文案为空 或
	•	文案越界被拦截
处理
V1 推荐
	•	该周信不插入时间流
	•	不向用户暴露“AI 失败”提示
原则
周信是惊喜，不是系统承诺。 生成失败不应污染主体验。

6. 轻组织层边界场景

6.1 月份胶囊未达到触发阈值
触发条件
	•	用户普通慢速滚动
	•	速度未超过 1500pt/s
处理
	•	月份胶囊不出现
原则
月份胶囊是时间穿梭工具，不是常驻辅助条。

6.2 数据只跨 1 个月
触发条件
	•	虽然有内容
	•	但所有内容都在同一月份内
处理
	•	不触发月份胶囊首次发现提示
	•	即使快滚，也可不出现或出现意义极低，应由实现层判断是否禁用
原则
只有在“多月内容”形成后，时间穿梭才有价值。

6.3 首次发现提示已出现过
触发条件
	•	用户已经看过一次 拖动这里穿梭时间
处理
	•	终生不再自动出现
	•	不因重装前端组件重复出现（需依赖本地持久状态）

6.4 过滤器吸顶与月份胶囊同时出现
触发条件
	•	用户快速滚动
	•	同时过滤器进入吸顶态
处理
	•	两者允许同时存在
	•	但月份胶囊优先靠右，过滤器保持顶部
	•	不互相遮挡
	•	视觉层级必须清楚

7. 输入流边界场景

7.1 空白关闭
触发条件
	•	无图
	•	无文字
	•	未打星
处理
	•	直接关闭
	•	不弹确认

7.2 有内容关闭
触发条件
满足任一：
	•	有图
	•	有文字
	•	已打星
处理
	•	弹出放弃确认
文案
	•	标题：放弃这条记录？
	•	次级说明：还没有保存。

7.3 保存失败
触发条件
	•	本地写入失败
	•	图片处理失败
	•	其他持久化失败
处理
	•	保留当前草稿
	•	停留在输入页
	•	显示轻错误文案： 没有保存成功，请再试一次。
操作
	•	再试一次
	•	返回编辑

7.4 已选图片后又删除图片
触发条件
	•	用户先选图
	•	后手动删除
处理
若有文字
	•	回到文字编辑态
	•	可保存
若无文字
	•	回到空白态或仅打星态
	•	若只有打星，仍不可保存

7.5 只输入空格或换行
触发条件
	•	note 存在
	•	但 trim 后为空
处理
	•	视为无文字
	•	不可作为合法保存依据

8. 滚动与阅读边界场景

8.1 快速滚动时误触卡片
触发条件
	•	用户处于 fast scrolling
	•	手指滑过卡片区域
处理
	•	优先视为滚动
	•	不应轻易触发周信展开或卡片点击
原则
快速浏览模式下，优先保护滚动意图。

8.2 周信展开时滚动列表
处理建议
V1 需统一一种模式：
方案 A：Inline 展开
	•	列表继续存在
	•	但展开内容要稳定，不因轻微滚动塌陷
方案 B：Modal 展开
	•	与列表分离
	•	关闭后回到原位置
V1 建议二选一固定，不混搭。

8.3 切换过滤器后滚动位置
处理建议
	•	切换过滤器后回到该过滤列表顶部
	•	不保留旧过滤器的滚动位置
原则
珍藏过滤更像“阅读视角切换”，不是工作区标签页。

9. 深色模式边界

9.1 里程碑底色过轻不可见
处理
	•	使用已定 8% 透明度
	•	若真机仍过弱，应优先微调底色，不调整正文可读性

9.2 星标颜色在深色背景发灰
处理
	•	可微调陶土色亮度
	•	但不可发光、不可变金色

9.3 周信卡片层级不清
处理
	•	优先通过背景层级和留白解决
	•	不通过重阴影和重描边解决

10. 删除 / 撤销边界

10.1 删除最后一条普通内容
处理
	•	页面重新进入 empty 或仅剩周信态
	•	若仍有周信，则不是完整空态

10.2 删除某条里程碑内容
处理
	•	从 starredMoments 过滤结果中立即移除
	•	若因此没有任何里程碑，则进入星标空态： 还没有被点亮的时刻。

10.3 撤销刚发布的内容
处理
	•	内容从时间流移除
	•	若它是本月唯一内容，月份锚点需要重算
	•	若它影响本周密度，不要求实时重算周信（V1 可延后到下次刷新）
原则
V1 不要求周信实时回滚重生成。

11. 不应出现的错误体验
Codex 实现时必须避免：
	1	图片加载失败导致整条卡片消失
	2	低内容态自动改成双列
	3	快速滚动时不断误点开卡片
	4	月份胶囊在慢速滚动时频繁出现
	5	首次提示重复出现
	6	只有打星也能保存
	7	空白关闭仍然弹确认
	8	周信展开后切换过滤器出现残影
	9	内容少时 AI 强行写长文
	10	深色模式里里程碑看不见或变脏

12. QA 验收 Checklist
页面空态
	•	0 内容空态正确
	•	低内容态仍保持单列
	•	各过滤器空态文案正确
内容异常
	•	图片失败可回退
	•	纯图可显示
	•	纯文可显示
	•	只有打星不可保存
周信边界
	•	静默周生成窄文本卡
	•	普通周内容不足时可降级
	•	高内容密度周无里程碑也可成立
	•	周信展开后切换过滤器不残留
轻组织层
	•	快滚才出现月份胶囊
	•	数据不足两个月不提示
	•	首次提示只出现一次
输入流
	•	空白关闭不提示
	•	有内容关闭才确认
	•	空白字符不算有效内容
	•	保存失败后草稿保留
