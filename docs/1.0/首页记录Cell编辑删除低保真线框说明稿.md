# 1. 文档目标

把首页记录 Cell 的编辑与删除能力，从抽象交互流程进一步落成可执行的页面骨架说明。

本稿只回答以下问题：

1. 页面上各层分别出现在哪里
2. 哪些状态会覆盖首页，哪些状态只在局部变化
3. 各种弹层、Toast、Sheet 的视觉层级如何排序
4. 设计稿与开发实现最少需要哪些页面状态

本稿不定义：

1. 最终视觉细节
2. 具体字号、间距数值
3. 最终图标样式
4. 完整组件 token

---

# 2. 使用方式

本稿为低保真结构说明，不追求像素级还原。

阅读规则：

1. `WF-xx` 表示一个独立线框页面或状态
2. 方框表示结构区域
3. 文案仅为占位示例
4. 同一页面中，越靠下的说明越接近用户操作重点

---

# 3. 总体交互原则

首页记录 Cell 编辑删除方案保持以下原则：

1. 默认看到的是内容，不是操作控件
2. 编辑是主动作，删除是低频动作
3. 删除必须先轻确认，再给撤销
4. 所有管理动作都不应破坏首页的安静感

---

# 4. 页面清单

本稿建议至少产出以下线框状态：

1. `WF-01` 首页默认时间线态
2. `WF-02` Cell 点按进入编辑前的按压态
3. `WF-03` Cell 长按菜单态
4. `WF-04` 删除确认 Sheet 态
5. `WF-05` 编辑成功后的轻反馈态
6. `WF-06` 删除成功后的 Undo Toast 态
7. `WF-07` 奶量记录编辑 Sheet
8. `WF-08` 尿布记录编辑 Sheet
9. `WF-09` 睡眠记录编辑 Sheet
10. `WF-10` 辅食记录编辑 Sheet

---

# 5. 线框总层级

在首页记录模块中，编辑与删除相关能力的页面层级顺序如下：

1. `L1` 首页主内容层
2. `L2` Cell 触控反馈层
3. `L3` Context Menu 层
4. `L4` 删除确认 Sheet 层
5. `L5` 编辑 Sheet 层
6. `L6` 结果反馈 Toast 层

说明：

1. `Context Menu` 与 `Sheet` 不会同时存在
2. `Undo Toast` 不覆盖 `Sheet`
3. 当 `Sheet` 打开时，首页底部胶囊仍处于背景层，不作为当前焦点

# 7. WF-02 Cell 点按按压态

## 7.1 目标

展示用户点按记录 Cell 后、编辑 Sheet 拉起前的瞬时反馈。

## 7.2 结构线框

```text
+----------------------------------------------+
| [icon]  120ml 配方奶                  09:24 |
|        可选副文案                           |
+----------------------------------------------+
        ↓
  卡片轻微压暗 / 轻微缩放
  约 80ms - 120ms 内进入编辑 Sheet
```

## 7.3 说明

1. 该态是瞬时态，不单独停留
2. 只为告诉用户“我点中了这条记录”
3. 不出现局部按钮，不展开卡片

# 8. WF-03 Cell 长按菜单态

## 8.1 目标

展示用户长按某条记录后，系统弹出的次级操作菜单。

## 8.2 结构线框

```text
背景：首页时间线轻微失焦

      +----------------------------------+
      | [icon] 120ml 配方奶       09:24 |
      +----------------------------------+

              +------------------+
              | 编辑记录         |
              | 删除记录         |
              +------------------+
```

## 8.3 说明

1. 当前被长按的 Cell 应作为上下文对象出现
2. 菜单只保留两项
3. 不要把菜单做成长操作面板

## 8.4 菜单项规范

1. 第一项：`编辑记录`
2. 第二项：`删除记录`

## 9.2 结构线框

         +----------------------------------+

         +----------------------------------+

# 16. 页面切换关系

各线框页面的主要流转关系如下：

```text
WF-01 首页默认态
  ├─ 点按 Cell ─────────────→ WF-07 / WF-08 / WF-09 / WF-10
  └─ 长按 Cell ─────────────→ WF-03

WF-03 长按菜单态
  ├─ 点击 编辑记录 ─────────→ WF-07 / WF-08 / WF-09 / WF-10
  └─ 点击 删除记录 ─────────→ WF-04

WF-04 删除确认 Sheet
  ├─ 点击 取消 ────────────→ WF-01
  └─ 点击 删除 ────────────→ WF-06

WF-07 / WF-08 / WF-09 / WF-10 编辑 Sheet
  ├─ 点击 关闭 ────────────→ WF-01
  └─ 点击 保存修改 ─────────→ WF-05

WF-06 Undo Toast
  ├─ 点击 撤销 ────────────→ WF-01
  └─ 4 秒自动消失 ─────────→ WF-01
```

# 17. 层级遮挡规则

## 17.1 首页默认态

1. 顶部导航可见
2. 时间线可滚动
3. 底部悬浮胶囊可见

## 17.2 Context Menu 打开时

1. 首页仍可见，但退为背景层
2. 胶囊不可成为当前交互焦点
3. 不显示 Undo Toast

## 17.3 删除确认 Sheet 打开时

1. 首页内容保留为背景层
2. 胶囊不可点击
3. 不叠加其他结果 Toast

## 17.4 编辑 Sheet 打开时

1. 编辑 Sheet 成为唯一焦点层
2. 首页所有 Cell 均不可交互
3. 结果 Toast 不应在编辑中途弹出

## 17.5 Toast 显示时

1. Toast 位于首页内容之上
2. Toast 位于底部悬浮胶囊之上
3. Toast 不遮挡顶部导航

# 18. 错误态线框建议

本稿不要求把所有错误态逐个画成独立页面，但设计稿中至少应补齐以下两种轻错误提示：

## 18.1 编辑失败

```text
+----------------------------------+
| 这次修改没有保存成功，请再试一次。 |
+----------------------------------+
```

## 18.2 删除失败

```text
+----------------------------------+
| 这次删除没有成功，请再试一次。     |
+----------------------------------+
```

说明：

1. 错误反馈仍走轻提示，不走高压弹窗
2. 不使用刺眼亮红色

# 19. 设计落稿提醒

设计细稿阶段，建议保持以下约束：

1. 首页默认态不出现任何常驻管理按钮
2. Context Menu 项数不得超过 2 项
3. 删除确认必须是底部 Sheet，不改成 Alert
4. 四类编辑 Sheet 应共享统一头部与底部主按钮逻辑
5. 所有记录类型的时间字段都应放在首屏可见区域
