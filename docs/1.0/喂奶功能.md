# 《初长》V1 喂奶模块开发说明（Codex 参照版）

## 1. 目标

实现首页「记录」页中的喂奶记录模块，采用半屏 Bottom Sheet 交互，支持：

- 亲喂计时
- 瓶喂记录
- 同一次记录中的混合喂养

本模块必须满足以下原则：

- 不弹系统数字键盘
- 不为混合喂养单独做第三个 Tab
- 亲喂和瓶喂共用一份草稿状态
- 切换 Tab 时状态不丢失
- 计时在切 Tab / 退后台 / 回前台后仍然准确
- 提交后首页只生成一条统一喂奶记录

---

## 2. 交付范围

### V1 必做
- Bottom Sheet：`MilkLoggingSheet`
- 顶部纯文本切换：`亲喂计时 | 瓶喂记录`
- 亲喂左右计时器
- 瓶喂预设 + stepper
- 动态提交按钮
- 首页喂奶时间线卡片
- 混合喂养支持
- 后台恢复后计时准确

### V1 不做
- 不做系统通知
- 不做自动识别喂奶开始/结束
- 不做复杂统计页
- 不做医学建议
- 不做同龄比较
- 不做独立“混合喂养”Tab
- 不做键盘输入
- 不做草稿自动保存

---

## 3. 页面结构

### 组件名称
`MilkLoggingSheet`

### 结构要求
- 使用 Bottom Sheet / 半屏弹窗
- 顶部为极简文本切换器
- 中间内容区根据当前 Tab 展示：
  - 亲喂计时 UI
  - 瓶喂记录 UI
- 底部固定一个动态提交按钮

### 重要约束
- 所有状态必须定义在 `MilkLoggingSheet` 顶层
- 子视图只接收 binding 或 view model，不得各自持有独立真值状态
- 切换 Tab 后不能丢失亲喂时间和瓶喂 ml

## 4. 顶层状态设计

### 推荐状态字段

```swift
enum NursingSide {
    case left
    case right
}

struct FeedingDraft {
    var leftAccumulatedSeconds: Int = 0
    var rightAccumulatedSeconds: Int = 0
    var activeSide: NursingSide? = nil
    var activeStartDate: Date? = nil
    var bottleAmountMl: Int = 0
}
````

如果直接用 `@State`，至少需要这些字段：

```swift
@State private var leftAccumulatedSeconds: Int = 0
@State private var rightAccumulatedSeconds: Int = 0
@State private var activeSide: NursingSide? = nil
@State private var activeStartDate: Date? = nil
@State private var bottleAmountMl: Int = 0
@State private var selectedTab: MilkTab = .nursing
```

### 设计原则

* 左右亲喂时间存累计值，不存“最后一段”
* 当前运行状态通过 `activeSide + activeStartDate` 表示
* 瓶喂 ml 独立存储
* 整个弹窗只维护一份 draft

* `亲喂计时`

## 6. 亲喂计时器功能规范

### 布局

* 左右两个大块：

  * 左：`左 L`
  * 右：`右 R`
* 平分宽度
* 高度约 200pt

### 默认状态

* 背景：`Theme.background`
* 文字：`Theme.textPrimary`

### 激活状态

* 当前计时侧：

  * 背景：`Theme.sageGreen`
  * 文字：白色
* 另一侧：

  * 自动暂停
  * 恢复默认状态

### 时间显示

* 使用 `.monospacedDigit()`
* 示例：`05:20`

### 行为规则

#### 点击未激活左侧

1. 触发 `.medium` haptic
2. 若右侧正在计时，先结算右侧累计时间并暂停
3. 左侧开始计时
4. 设置：

   * `activeSide = .left`
   * `activeStartDate = now`

#### 点击未激活右侧

1. 触发 `.medium` haptic
2. 若左侧正在计时，先结算左侧累计时间并暂停
3. 右侧开始计时
4. 设置：

   * `activeSide = .right`
   * `activeStartDate = now`

#### 点击当前激活侧

1. 触发 `.medium` haptic
2. 结算当前侧累计时间
3. 清空运行状态：

   * `activeSide = nil`
   * `activeStartDate = nil`

### 结算逻辑

不要用“每秒 +1”作为唯一真值。
必须使用：

* `leftAccumulatedSeconds`
* `rightAccumulatedSeconds`
* `activeStartDate`

#### 推荐结算函数

```swift
func pauseActiveSide(now: Date = Date()) {
    guard let side = activeSide, let start = activeStartDate else { return }
    let delta = max(0, Int(now.timeIntervalSince(start)))

    switch side {
    case .left:
        leftAccumulatedSeconds += delta
    case .right:
        rightAccumulatedSeconds += delta
    }

    activeSide = nil
    activeStartDate = nil
}
```

#### 推荐实时展示函数

```swift
func displayedSeconds(for side: NursingSide, now: Date = Date()) -> Int {
    let base: Int
    switch side {
    case .left:
        base = leftAccumulatedSeconds
    case .right:
        base = rightAccumulatedSeconds
    }

    guard activeSide == side, let start = activeStartDate else {
        return base
    }

    return base + max(0, Int(now.timeIntervalSince(start)))
}
```

### 强约束

* 任意时刻只允许一个侧运行
* 切换 Tab 不停止计时
* 退后台后回前台，时间仍正确
* 计时精度依赖 `Date` 计算，不依赖前台定时器累加

## 7. 瓶喂记录功能规范

### 预设值

2x2 网格：

* 90ml
* 120ml
* 150ml
* 180ml

### 点击行为

* 触发 `.light` haptic
* 直接设置 `bottleAmountMl = presetValue`

### 高亮规则

* 当 `bottleAmountMl` 恰好等于某预设值时，该项高亮
* 如果 stepper 改成非预设值，则无任何预设高亮

### Stepper 规则

布局：

* 左减号按钮
* 中间当前 ml 文案
* 右加号按钮

### Stepper 约束

* 每次增减 10ml
* 最小值：0ml
* 最大值：建议 300ml（如有业务决定，也可改为 360ml）
* 点击触发 `.soft` haptic

### 示例逻辑

```swift
let bottleMin = 0
let bottleMax = 300
let bottleStep = 10

func increaseBottle() {
    bottleAmountMl = min(bottleAmountMl + bottleStep, bottleMax)
}

func decreaseBottle() {
    bottleAmountMl = max(bottleAmountMl - bottleStep, bottleMin)
}
```

### 关键约束

* 禁止使用 `TextField`
* 禁止弹系统数字键盘

---

## 8. 混合喂养规则

### 核心定义

混合喂养不是第三种模式，而是同一份 draft 中同时存在：

* 亲喂累计时间
* 瓶喂 ml

### 必须遵守

* 不新增“混合喂养”Tab
* 通过在两个现有 Tab 之间来回切换静默实现
* 弹窗关闭前不生成部分记录
* 只有点击提交时，才统一生成一条最终记录

### 示例流程

用户可以这样操作：

1. 左侧亲喂 8 分钟
2. 切到瓶喂记录 60ml
3. 再切回右侧亲喂 5 分钟
4. 点击提交

最终生成：

* 左 8 分钟
* 右 5 分钟
* 瓶喂 60ml

首页显示一条混合记录。

## 9. 动态提交按钮规范

### 样式

* 固定在 Bottom Sheet 底部安全区上方
* 全宽
* 高度 56pt
* 背景：`Theme.textPrimary`
* 文字：白色

### 点击行为

* 触发 `.medium` haptic
* 若有正在运行中的亲喂侧，先自动暂停并结算
* 创建一条最终喂奶记录
* 关闭弹窗
* 清空本次草稿

### 可点击条件

* 当且仅当以下任一成立时可点击：

  * `totalNursingSeconds > 0`
  * `bottleAmountMl > 0`

### 禁止

* 禁止提交空记录

### 动态文案逻辑

先定义：

```swift
let totalNursingSeconds = leftAccumulatedSeconds + rightAccumulatedSeconds
let totalNursingMinutes = floorMinutes(totalNursingSeconds)
```

#### 混合喂养

当 `totalNursingSeconds > 0 && bottleAmountMl > 0`

文案：

* `记录 X分钟亲喂 + Yml 瓶喂`

#### 仅亲喂

当 `totalNursingSeconds > 0 && bottleAmountMl == 0`

文案可选其一：

* `记录 X分钟亲喂`
* 或 `完成记录`

推荐优先：

* `记录 X分钟亲喂`

#### 仅瓶喂

当 `totalNursingSeconds == 0 && bottleAmountMl > 0`

文案可选其一：

* `记录 Yml 瓶喂`
* 或 `完成记录`

推荐优先：

* `记录 Yml 瓶喂`

#### 空状态

当两者都为 0：

* 按钮禁用
* 文案可显示 `完成记录`
* 但不可点击

## 10. 时间换算规则

### 存储规则

* 所有亲喂时间底层一律存秒

### 展示规则

* 展示时统一转换为分钟
* 推荐使用**向下取整**

```swift
func floorMinutes(_ seconds: Int) -> Int {
    max(0, seconds / 60)
}
```

### 原因

* 更稳妥
* 避免出现“显示比真实更大”的感受
* 首页主标题与副标题更容易保持一致

### 强约束

主标题和副标题必须使用同一套换算规则，避免：

* 主标题：25分钟
* 副标题：左14m · 右10m
  合计变成 24m

## 11. 弹窗生命周期规则

### 打开弹窗

* 每次打开默认新建一份空草稿
* 不保留上次未提交内容

### 关闭弹窗（用户主动 dismiss）

推荐 V1 规则：

1. 若有正在运行中的侧，先自动暂停结算
2. 不自动保存
3. 丢弃当前 draft
4. 下次打开重新开始

### 不允许

* 关闭后继续暗中计时
* 自动保存未确认记录
* 下次打开残留旧 draft（V1 先不要做草稿恢复）

## 12. App 生命周期处理

### 必须处理的场景

* 切换亲喂/瓶喂 Tab
* App 进入后台
* App 回到前台
* Sheet dismiss
* 视图重绘

### 推荐实现

监听 scene phase：

```swift
@Environment(\.scenePhase) private var scenePhase
```

当进入后台时：

* 不必强制暂停
* 但必须保证回到前台后，展示值依旧通过 `Date` 补算正确

当提交或关闭时：

* 才真正结算并写入累计值 / 或丢弃草稿

### 结论

后台期间不依赖定时器继续跑，
只依赖 `Date()` 差值恢复真实耗时。

## 13. 数据模型

### 最终记录模型

建议统一为一个结构，不要拆成 nursing / bottle / mixed 三种类型。

```swift
struct FeedingRecord: Identifiable, Codable {
    let id: UUID
    let createdAt: Date
    let leftSeconds: Int
    let rightSeconds: Int
    let bottleAmountMl: Int
}
```

### 推导字段

```swift
extension FeedingRecord {
    var totalNursingSeconds: Int {
        leftSeconds + rightSeconds
    }

    var totalNursingMinutes: Int {
        totalNursingSeconds / 60
    }

    var isBottleOnly: Bool {
        totalNursingSeconds == 0 && bottleAmountMl > 0
    }

    var isNursingOnly: Bool {
        totalNursingSeconds > 0 && bottleAmountMl == 0
    }

    var isMixed: Bool {
        totalNursingSeconds > 0 && bottleAmountMl > 0
    }
}
```

### 为什么不要单独存 type = mixed

因为 mixed 只是数据组合，不是独立输入模式。
统一模型更容易支持：

* 首页展示
* 未来编辑
* 未来统计

## 14. 首页时间线卡片生成规则

提交后首页只生成一条喂奶记录卡片。

### 图标与底座

* 图标：`Theme.textPrimary` 线稿
* 背景圆底：`Theme.sageGreen.opacity(0.15)`

### 文案规则

#### 纯瓶喂

主标题：

* `120ml 瓶喂`

#### 纯亲喂

主标题：

* `亲喂 25分钟`

副标题：

* `左 15m · 右 10m`

#### 混合喂养

主标题：

* `亲喂 25分钟 + 60ml 瓶喂`

### 文案原则

* 只描述事实
* 不加判断
* 不加建议
* 不加奖励性语言
* 不出现“足够/不足/优秀/偏少”等词

### 推荐文案生成函数

```swift
func makeFeedTitle(record: FeedingRecord) -> String {
    let totalMinutes = floorMinutes(record.totalNursingSeconds)

    if record.isMixed {
        return "亲喂 \(totalMinutes)分钟 + \(record.bottleAmountMl)ml 瓶喂"
    } else if record.isNursingOnly {
        return "亲喂 \(totalMinutes)分钟"
    } else {
        return "\(record.bottleAmountMl)ml 瓶喂"
    }
}

func makeFeedSubtitle(record: FeedingRecord) -> String? {
    guard record.isNursingOnly || record.isMixed else { return nil }

    let leftMinutes = floorMinutes(record.leftSeconds)
    let rightMinutes = floorMinutes(record.rightSeconds)

    if leftMinutes == 0 && rightMinutes == 0 { return nil }
    return "左 \(leftMinutes)m · 右 \(rightMinutes)m"
}
```

> 注：混合喂养是否显示副标题可自行决定。
> 若首页希望更简洁，可只在纯亲喂时显示副标题。

## 17. 关键实现建议

### 1. 不要把定时器当真值

可以用 UI timer 触发界面刷新，但不能用它来定义累计时长。
真值必须来自：

* 已累计秒数
* 开始时间戳
* 当前时间

### 2. 不要把状态拆进 Tab 子视图

子视图只负责展示。
所有真实数据必须留在 `MilkLoggingSheet` 顶层或其 view model。

### 3. 提交前一定先 pause

如果用户在计时中点击提交，必须先结算当前侧再创建记录。

### 4. 首页不要存死文案

首页卡片标题、副标题用模型实时推导，便于未来编辑和统计。
