# 初长 V1.0｜珍藏模块数据契约 / 字段字典

**文档用途**：交给 Codex / iOS 开发 / 联调 / QA
**模块范围**：珍藏（照片、日记、里程碑、AI 周信）
**目标**：明确珍藏模块的数据结构、字段语义、存储边界与展示映射规则

---

## 1. 总体说明

珍藏模块消费两类核心数据：

1. **用户主动留下的记忆内容**

   * 照片
   * 日记
   * 里程碑标记

2. **系统按周生成的 AI 周信**

   * 静默周
   * 普通周
   * 高内容密度周

V1 的实现原则是：

* 用户内容与 AI 内容必须是两类不同对象
* 里程碑不是独立模块，而是用户内容的一种轻标记属性
* 周信不是消息通知，而是珍藏时间流中的一种特殊卡片

---

# 2. 顶层对象总览

珍藏模块建议至少定义以下对象：

* `MemoryEntry`
* `WeeklyLetter`
* `TreasureTimelineItem`
* `TreasureMonthAnchor`
* `TreasureFilter`
* `TreasureComposeDraft`

---

# 3. 用户内容对象：MemoryEntry

## 3.1 定义

```swift
struct MemoryEntry: Identifiable {
    let id: UUID
    let createdAt: Date
    let ageInDays: Int?
    let imageLocalPath: String?
    let note: String?
    let isMilestone: Bool
}
```

---

## 3.2 字段说明

### `id`

* 类型：`UUID`
* 语义：用户记忆内容唯一标识
* 必填：是
* 用途：

  * 列表 diff
  * 删除 / 撤销
  * 定位月份锚点

---

### `createdAt`

* 类型：`Date`
* 语义：该条内容创建时间
* 必填：是
* 用途：

  * 时间流排序
  * 月份归属
  * 时间戳展示
  * 周信生成归档

---

### `ageInDays`

* 类型：`Int?`
* 语义：宝宝在该内容创建当天的天数
* 必填：否
* 用途：

  * 时间戳展示中的 `460天`
* 说明：

  * 若系统已具备宝宝出生日期，可在展示层计算
  * 若 V1 暂未稳定持有出生日期，也可在写入时持久化

---

### `imageLocalPath`

* 类型：`String?`
* 语义：本地图片路径
* 必填：否
* 用途：

  * 卡片照片显示
* 约束：

  * V1 每条 `MemoryEntry` 最多 1 张图片
* 禁止：

  * 不使用远程 URL 作为 V1 主语义
  * 不扩展为多图数组

---

### `note`

* 类型：`String?`
* 语义：用户输入的文字内容
* 必填：否
* 用途：

  * 日记正文展示
  * 里程碑轻标题规则判断
  * AI 周信输入素材

---

### `isMilestone`

* 类型：`Bool`
* 语义：该条内容是否被用户主动标记为里程碑
* 必填：是
* 默认值：`false`
* 用途：

  * 卡片高亮
  * 过滤到“星标时刻”
  * 周信密度判断

---

## 3.3 保存合法性规则

一条 `MemoryEntry` 合法保存的最小条件：

* `imageLocalPath != nil`
  或
* `note` 非空（去掉空白后）

### 不合法情况

* 没有图片
* 没有文字
* 只有 `isMilestone = true`

即：

```swift
if imageLocalPath == nil && note.trimmed.isEmpty {
    invalid
}
```

---

## 3.4 展示映射

### 有图 + 有文

渲染为完整 `TreasureMemoryCard` / `TreasureMilestoneCard`

### 只有图

渲染为图片卡 + 可无正文

### 只有文

渲染为纯文字记忆卡

### isMilestone = true

在基础卡上叠加：

* 轻底色
* 星标
* 第一行轻强化规则

# 4. AI 周信对象：WeeklyLetter

## 4.1 定义

```swift
enum WeeklyLetterDensity {
    case silent
    case normal
    case dense
}

struct WeeklyLetter: Identifiable {
    let id: UUID
    let weekStart: Date
    let weekEnd: Date
    let density: WeeklyLetterDensity
    let collapsedText: String
    let expandedText: String
    let generatedAt: Date
}
```

---

## 4.2 字段说明

### `id`

* 类型：`UUID`
* 语义：每封周信唯一标识
* 必填：是

---

### `weekStart`

* 类型：`Date`
* 语义：该周开始时间
* 必填：是

### `weekEnd`

* 类型：`Date`
* 语义：该周结束时间
* 必填：是

### `density`

* 类型：`WeeklyLetterDensity`
* 必填：是
* 合法值：

  * `silent`
  * `normal`
  * `dense`

---

### `collapsedText`

* 类型：`String`
* 语义：时间流中收起态文案
* 必填：是
* 约束：

  * 静默周：极短
  * 普通周 / 高内容周：信封外一句话

---

### `expandedText`

* 类型：`String`
* 语义：展开后完整文案
* 必填：是
* 约束：

  * `silent`：短于 30 字上下
  * `normal`：≤100 字
  * `dense`：≤250 字

---

### `generatedAt`

* 类型：`Date`
* 语义：周信生成时间
* 必填：是

---

## 4.3 展示映射

### `density = silent`

渲染为：

* `WeeklyLetterSilentCard`

### `density = normal`

渲染为：

* `WeeklyLetterNormalCard`

### `density = dense`

渲染为：

* `WeeklyLetterDenseCard`

---

## 4.4 生成规则映射

### `silent`

适用于：

* 无图片
* 无日记
* 无里程碑
* 珍藏内容极少或没有

### `normal`

适用于：

* 有 1–4 条有效珍藏内容
* 无里程碑
* 内容达到最小门槛

### `dense`

适用于任一：

* 至少 1 个里程碑
* 内容数量 ≥ 5
* 明显有连续故事性内容

# 5. 时间流统一展示对象：TreasureTimelineItem

为了让列表不直接消费两种不同数据模型，建议增加统一展示对象。

## 5.1 定义

```swift
enum TreasureTimelineItemType {
    case memory
    case milestone
    case weeklyLetterSilent
    case weeklyLetterNormal
    case weeklyLetterDense
}

struct TreasureTimelineItem: Identifiable {
    let id: UUID
    let type: TreasureTimelineItemType
    let createdAt: Date
    let monthKey: String
    let sourceMemoryEntry: MemoryEntry?
    let sourceWeeklyLetter: WeeklyLetter?
}
```

---

## 5.2 作用

* 时间流统一排序
* 统一按月份归档
* 统一 diff / 刷新
* 统一过滤逻辑

---

## 5.3 `type` 映射规则

### MemoryEntry 映射

* `isMilestone = false` -> `.memory`
* `isMilestone = true` -> `.milestone`

### WeeklyLetter 映射

* `density = silent` -> `.weeklyLetterSilent`
* `density = normal` -> `.weeklyLetterNormal`
* `density = dense` -> `.weeklyLetterDense`

---

## 5.4 `monthKey`

* 类型：`String`
* 语义：月份归档键
* 格式建议：`2024-10`
* 用途：

  * 月份胶囊吸附
  * 月份分组
  * 时间穿梭定位

---

# 6. 月份锚点对象：TreasureMonthAnchor

## 6.1 定义

```swift
struct TreasureMonthAnchor: Identifiable {
    let id: String            // same as monthKey
    let monthKey: String      // e.g. 2024-10
    let displayText: String   // e.g. 2024 · 10月
    let firstTimelineItemID: UUID
}
```

---

## 6.2 用途

* 月份胶囊显示文案
* 月份拖动时定位到首条内容
* 一次性提示逻辑判断数据是否跨 2 个月

---

## 6.3 生成规则

每个 `monthKey` 只生成一个 anchor，指向该月第一条 `TreasureTimelineItem`

# 7. 过滤对象：TreasureFilter

## 7.1 定义

```swift
enum TreasureFilter {
    case allMemories
    case starredMoments
    case timeLetters
}
```

---

## 7.2 数据筛选规则

### `.allMemories`

包含：

* `.memory`
* `.milestone`
* `.weeklyLetterSilent`
* `.weeklyLetterNormal`
* `.weeklyLetterDense`

### `.starredMoments`

仅包含：

* `.milestone`

### `.timeLetters`

仅包含：

* `.weeklyLetterSilent`
* `.weeklyLetterNormal`
* `.weeklyLetterDense`

# 8. 输入草稿对象：TreasureComposeDraft

## 8.1 定义

```swift
struct TreasureComposeDraft {
    var imageLocalPath: String?
    var note: String
    var isMilestone: Bool
}
```

---

## 8.2 默认值

```swift
TreasureComposeDraft(
    imageLocalPath: nil,
    note: "",
    isMilestone: false
)
```

---

## 8.3 草稿合法性

### 可保存

* 有图
  或
* 有文字

### 不可保存

* 无图
* 无文字
* 仅 `isMilestone = true`

---

## 8.4 草稿退出判断

### 无内容

直接退出，不提示

### 有内容

出现放弃确认

“有内容”定义为任一成立：

* `imageLocalPath != nil`
* `note.trimmed.isEmpty == false`
* `isMilestone == true`

> 注意：
> 虽然只有 `isMilestone = true` 不能保存，但它依然算“用户已操作过”，关闭时应提示是否放弃。

# 9. 时间戳展示字段规则

珍藏卡片顶部元信息行使用以下展示逻辑：

## 9.1 显示格式

`OCT 12 · 460天`

### 前半段：日期

* 建议使用固定英文月份缩写 + 日期
* 如：`OCT 12`

### 后半段：宝宝天数

* 若有 `ageInDays`，则显示
* 如：`460天`

### 无 `ageInDays`

仅显示日期部分：
`OCT 12`

---

## 9.2 月份胶囊显示格式

`2024 · 10月`

注意：

* 时间戳与月份胶囊可采用不同显示格式
* 但各自内部必须统一，不混用英文/月中文

---

# 10. 里程碑轻标题判断辅助字段（展示层）

不建议把“是否强化第一行”写入持久层。
建议在展示层临时计算。

## 10.1 辅助结果对象

```swift
enum TreasureFirstLineStyle {
    case regular
    case enhanced
}
```

---

## 10.2 计算规则

输入：`note`

输出：`regular` 或 `enhanced`

### enhanced

满足任一：

* 第一行因用户主动换行而独立
* 第一行以 `。！？～` 结尾，且 ≤18 个汉字

### regular

其他全部情况

# 11. AI 周信文案约束对象（可选）

如果你们希望把文案边界也结构化，可以补一个生成约束层对象：

```swift
enum WeeklyLetterTonePolicy {
    case descriptiveOnly
}
```

约束：

* 只描述，不评价
* 不出现健康结论
* 不出现同龄比较
* 不出现成就语言

V1 可不落库存储，但建议在 AI 生成服务层固定。

# 12. 数据排序规则

珍藏时间流统一按 `createdAt DESC` 排序。

## 12.1 MemoryEntry

按创建时间排序

## 12.2 WeeklyLetter

建议以 `weekEnd` 作为主要展示时间，再映射到 `createdAt`
确保其自然插入对应周的时间流位置

---

# 13. 删除 / 撤销规则

## 13.1 MemoryEntry

支持删除 / 撤销

## 13.2 WeeklyLetter

V1 不建议用户手动删除单周 AI 周信（若产品未定义相关能力）
若要隐藏，应作为后续策略，不在本轮实现

---

# 14. 边界规则

## 14.1 图片丢失

若 `imageLocalPath` 无效：

* 不隐藏整条 MemoryEntry
* 回退为文字卡
* 若连文字也无，则进入异常态，不应正常展示

## 14.2 只有图片

合法，允许发布

## 14.3 只有文字

合法，允许发布

## 14.4 只有打星

不允许发布

## 14.5 静默周

不是“不生成对象”，而是生成 `WeeklyLetter(density: .silent)`

---

# 15. Codex 禁止事项

Codex 在实现数据层时不得：

1. 把里程碑做成独立 checklist 表
2. 把周信和普通记忆内容混成同一种原始对象
3. 把 `isMilestone = true` 当作可单独保存内容
4. 把第一行强化结果持久化成用户内容本体
5. 把月份胶囊文本格式做成多套混用
6. 把静默周省略不生成
7. 把 AI 周信当通知系统对象处理
