模块范围：珍藏（照片、日记、里程碑、AI 周信）
目标：定义页面状态、事件、切换结果与边界行为

1. 状态机设计原则

珍藏模块虽然视觉很安静，但内部状态不少。
状态设计必须遵守：

默认内容优先，工具退后
所有显隐都应有明确触发条件
不能因为动画或滚动状态导致组件错乱残留
周信、过滤器、月份胶囊、全屏输入页必须彼此解耦
2. 页面核心状态总览

珍藏模块至少包含以下状态域：

页面数据状态
过滤状态
滚动与轻组织层状态
月份胶囊状态
卡片状态
周信状态
全屏输入状态
3. 页面数据状态
3.1 TreasureDataState
enum TreasureDataState {
    case loading
    case empty
    case lowContent
    case ready
    case error
}
说明
loading：首次加载或切换过滤时的数据准备中
empty：完全无内容
lowContent：有内容但非常少（如 1–2 条）
ready：正常可浏览
error：数据异常或加载失败
切换规则
启动页面：loading -> empty/lowContent/ready/error
发布新内容后：通常进入 ready
删除最后一条后：进入 empty
4. 过滤状态
4.1 TreasureFilterState
enum TreasureFilterState {
    case allMemories
    case starredMoments
    case timeLetters
}
文案映射
allMemories -> 全部记忆
starredMoments -> 星标时刻
timeLetters -> 时光信笺
默认值

默认进入：
allMemories

切换结果
allMemories

显示：

普通日常卡
里程碑卡
AI 周信卡
starredMoments

只显示：

TreasureMilestoneCard
timeLetters

只显示：

TreasureWeeklyLetterCard
4.2 过滤切换事件
事件
tapAllMemories
tapStarredMoments
tapTimeLetters
状态变化

例如：

allMemories -> starredMoments
starredMoments -> timeLetters
UI 响应
更新当前过滤状态
时间流内容重新映射
保持页面仍在珍藏模块上下文中
过渡方式应轻，不是硬刷新
边界要求
切换过滤时，若有展开中的周信，需先收起或重建列表状态
不允许旧过滤状态的卡片残留
5. 滚动与轻组织层状态

珍藏模块的“轻组织层”包含：

顶部过滤条
月份胶囊

它们都与滚动强绑定。

5.1 TreasureScrollIntentState
enum TreasureScrollIntentState {
    case idle
    case readingDown
    case reversingUp
    case fastScrolling
    case monthScrubbing
}
说明
idle：静止
readingDown：用户正常向下浏览历史
reversingUp：用户明确向上回滚
fastScrolling：快速滚动，准备触发月份胶囊
monthScrubbing：用户按住月份胶囊进行时间穿梭
5.2 顶部过滤条显示状态
enum TreasureFilterBarVisibilityState {
    case inlineVisible
    case hiddenByScroll
    case pinnedVisible
}
状态说明
inlineVisible：过滤条位于内容顶部，正常可见
hiddenByScroll：随内容滚出屏幕
pinnedVisible：向上回滚后吸顶显示
触发逻辑
初始

默认：
inlineVisible

向下浏览

当用户向下浏览历史时：
inlineVisible -> hiddenByScroll

向上回滚

当检测到明确反向浏览意图：
hiddenByScroll -> pinnedVisible

回到顶部附近

当内容滚回页面顶部附近：
pinnedVisible -> inlineVisible

关键约束
不能因轻微抖动频繁切换
吸顶出现应有轻阈值，不是任何 1px 回滚都触发
6. 月份胶囊状态机
6.1 TreasureMonthScrubberState
enum TreasureMonthScrubberState {
    case hidden
    case appearing
    case visible
    case dragging
    case fading
    case onboardingNudge
}
状态说明
hidden：默认不可见
appearing：快滚触发后淡入
visible：可见但未被拖动
dragging：用户按住并拖动
fading：停止后延迟淡出
onboardingNudge：首次发现提示阶段
6.2 月份胶囊触发事件
事件
fastScrollDetected
scrollStopped
touchDownOnMonthScrubber
dragMonthScrubber
releaseMonthScrubber
firstDiscoveryEligible
状态流
普通快滚

hidden -> appearing -> visible -> fading -> hidden

拖动穿梭

visible -> dragging -> fading -> hidden

首次发现

hidden -> onboardingNudge -> hidden

6.3 月份胶囊显示规则
出现条件

满足：

快速滚动
速率 > 1500pt/s
当前数据已跨 2 个月以上，或页面已有多月内容
不出现条件
普通慢速拖拽
内容不足一个月
正在全屏输入页
当前正在展开周信全屏阅读（若采用 modal）
6.4 月份胶囊拖动规则
进入拖动

事件：
touchDownOnMonthScrubber

状态：
visible -> dragging

拖动中行为
进入“时间穿梭模式”
以月份锚点吸附跳转
主列表同步滚动
胶囊文案同步变化，例如：
2024 · 10月
2024 · 11月
释放后

事件：
releaseMonthScrubber

状态：
dragging -> fading -> hidden

列表定位规则

释放后停在：

对应月份的首条内容附近

不是：

月份中间随机位置
两个月交界处模糊停留
6.5 月份胶囊首次发现提示状态
触发条件

仅一次，且满足：

用户数据跨越 2 个月
首次进入珍藏页
之前从未看过提示
行为
胶囊淡入
轻微下移 30pt
伴随极轻微触感
显示旁侧气泡：拖动这里穿梭时间
停留 2 秒后淡出
永久记录为已提示
边界要求
此提示终生只出现一次
不得重复打扰
7. 珍藏卡片状态

珍藏页有三种卡片，不同卡片状态不同。

7.1 普通日常卡 TreasureMemoryCard
状态
enum TreasureMemoryCardState {
    case normal
    case imageMissing
    case textOnly
}
说明
normal：有图有文或有图无文
imageMissing：原本有图但加载失败
textOnly：无图，仅文字
处理规则
imageMissing
不隐藏整张卡
退回为文字卡
可显示极轻占位，不可显得损坏
7.2 里程碑卡 TreasureMilestoneCard
状态
enum TreasureMilestoneCardState {
    case normalMilestone
    case titleEnhanced
    case noEnhancedTitle
}
说明
normalMilestone：基本里程碑态
titleEnhanced：第一行满足轻强化规则
noEnhancedTitle：不满足轻强化条件，正文全 Regular
触发条件
titleEnhanced

满足任一：

用户主动换行
第一行句末有标点且 ≤18 汉字
noEnhancedTitle
第一行过长
无断句
连续碎碎念
7.3 周信卡 TreasureWeeklyLetterCard
状态
enum TreasureWeeklyLetterDensity {
    case silent
    case normal
    case dense
}

enum TreasureWeeklyLetterViewState {
    case collapsed
    case expandedInline
    case expandedModal
}
说明
silent / normal / dense：周信类型
collapsed：默认收起
expandedInline：在流内展开
expandedModal：半屏/全屏展开

V1 建议二选一：
要么统一 inline 展开，要么统一 modal 展开，不建议一类 inline、一类 modal 混搭。

8. AI 周信生成状态机
8.1 WeeklyLetterGenerationState
enum WeeklyLetterGenerationState {
    case notEligible
    case silentWeek
    case normalWeek
    case denseWeek
}
说明

这是“该周应该生成什么”的逻辑状态，不是 UI 展示状态。

8.2 生成规则
A. 静默周

触发条件：

无照片
无日记
无里程碑
仅基础数据或几乎没打开珍藏

结果：
notEligible -> silentWeek

B. 普通周

触发条件：

有 1–4 张照片或日记
无里程碑
达到最小有效内容门槛
不是只有一条极弱内容

结果：
notEligible -> normalWeek

C. 高内容密度周

触发条件满足任一：

至少 1 个星标里程碑
照片 / 日记 >= 5 条
有明显连续叙事内容

结果：
notEligible -> denseWeek

8.3 UI 形态映射
silentWeek
使用 WeeklyLetterSilentCard
normalWeek
使用 WeeklyLetterNormalCard
denseWeek
使用 WeeklyLetterDenseCard
8.4 周信展开事件
事件
tapWeeklyLetterCard
closeWeeklyLetter
switchFilterWhileLetterExpanded
状态流
collapsed -> expanded

用户点击卡片时展开

expanded -> collapsed

用户关闭后收起

过滤切换时

若周信处于展开态：

先关闭或销毁展开态
再切换过滤结果
边界要求
不允许“展开中的周信”残留到另一过滤器中
不允许静默周卡片点开后变成长信
9. “留住今天”输入流状态机
9.1 TreasureComposeState
enum TreasureComposeState {
    case closed
    case opening
    case editingEmpty
    case editingTextOnly
    case editingPhotoOnly
    case editingPhotoAndText
    case editingMilestone
    case confirmingDiscard
    case saving
    case failed
}
说明
closed：未打开
opening：全屏 modal 推起中
editingEmpty：空白态
editingTextOnly：只输入文字
editingPhotoOnly：只选图片
editingPhotoAndText：图文都有
editingMilestone：当前打开了里程碑开关
confirmingDiscard：用户想退出但已有内容
saving：保存中
failed：保存失败
9.2 事件表
事件
tapAddToday
selectPhoto
removePhoto
editText
toggleMilestoneOn
toggleMilestoneOff
tapCloseCompose
confirmDiscard
cancelDiscard
tapSaveCompose
saveSuccess
saveFailure
9.3 关键状态切换
打开

closed -> opening -> editingEmpty

输入文字

editingEmpty -> editingTextOnly

只选图片

editingEmpty -> editingPhotoOnly

图文并存

editingTextOnly -> editingPhotoAndText
或
editingPhotoOnly -> editingPhotoAndText

打星

任意编辑态下：

toggleMilestoneOn -> editingMilestone
toggleMilestoneOff -> 回到对应内容态
关闭时
若完全空白

直接：
editingEmpty -> closed

若已有内容

进入：
editing* -> confirmingDiscard

放弃

confirmingDiscard + confirmDiscard -> closed

继续编辑

confirmingDiscard + cancelDiscard -> 回到原编辑态

保存

editing* -> saving -> saveSuccess -> closed
或
editing* -> saving -> failed

9.4 保存合法性规则

允许保存的最小条件：

有文字
或
有照片

里程碑开关不能单独构成可保存内容。

即：

只有打星，没有文字和照片 -> 不可保存
10. 过滤、滚动、输入流之间的互斥关系
10.1 输入页打开时
顶部过滤器不响应
月份胶囊不可出现
时间流滚动冻结
10.2 快速滚动时
不触发卡片展开
主要目标是浏览与定位
10.3 过滤切换时
若有展开中的周信，需先关闭
若有月份胶囊可见，需先隐藏或重置

11.1 页面级

11.3 周信级
事件	前置状态	结果

11.4 输入流级
事件	前置状态	结果
tapAddToday	closed	opening -> editingEmpty

selectPhoto	editingEmpty	editingPhotoOnly

12. 开发禁止事项

Codex 在实现珍藏模块状态机时不得：

让月份胶囊慢速滚动也出现
让首次提示重复出现
让过滤器常驻霸占顶部空间
让里程碑变成独立任务系统
让周信每周固定同一形态
让静默周也变成完整信封
让全屏输入页无内容退出也弹确认
让“只有打星”也能保存
在过滤切换时残留展开中的周信
在输入页打开时仍能触发月份胶囊
13. 验收重点
过滤器
 下滚消失
 上滚吸顶
 切换后列表正确更新
月份胶囊
 仅快滚出现
 可拖动穿梭
 松手定位到月份锚点
 首次提示只出现一次
里程碑
 打星才形成里程碑
 第一行强化规则正确执行
 底色和星标都足够轻
周信
 静默周 / 普通周 / 高内容密度周分层正确
 展开与收起稳定
 过滤切换不残留状态
输入流
 图 / 文任一存在即可保存
 空白关闭不提示
 有内容关闭才确认
 仅打星不能保存
