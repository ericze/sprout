# 1. 文档目标

在低保真线框基础上，进一步明确首页记录 Cell 编辑删除能力的视觉气质、组件层级、排版、颜色、间距、动效与文案呈现方式。

本稿回答以下问题：

1. 这套交互在视觉上应该长什么样
2. 默认态、编辑态、删除态分别呈现什么气质
3. 各类反馈层如何与首页、Bottom Sheet 和时间线卡片风格统一
4. 设计细稿与 SwiftUI 实现需要共同遵守哪些视觉约束

本稿不解决：

1. 具体代码实现
2. Figma 节点命名规范
3. 国际化逐字拷贝验收

---

# 2. 设计关键词

该能力必须延续《初长》的整体气质：

1. 极简
2. 温暖
3. 编辑感
4. Apple 原生
5. Invisible UI

在视觉执行上，尤其要避免以下倾向：

1. 工具化后台感
2. 管理面板感
3. 鲜艳危险提示
4. 过度强调“删除”这件事
5. 复杂控件堆叠

---

# 3. 视觉总原则

首页记录 Cell 的管理能力必须遵循以下四个视觉原则：

## 3.1 内容优先

用户首先看到的是记录内容，而不是操作入口。

## 3.2 控件隐退

管理能力默认隐藏在点按与长按中，不通过常驻按钮抢占视线。

## 3.3 风险降噪

删除是风险动作，但表达方式必须克制，不借助刺眼红色制造紧张。

## 3.4 系统一致

所有新增页面状态都应与现有 `AppTheme`、首页卡片、Undo Toast、Sheet 结构保持一致。

# 4. 视觉基线

本稿默认沿用当前设计系统 token。

## 4.1 颜色

1. 页面背景：`AppTheme.Colors.background`
2. 卡片背景：`AppTheme.Colors.cardBackground`
3. 主文本：`AppTheme.Colors.primaryText`
4. 次文本：`AppTheme.Colors.secondaryText`
5. 辅助文本：`AppTheme.Colors.tertiaryText`
6. 强调色：`AppTheme.Colors.accent`
7. 重要高亮仅限特殊场景，不用于删除流程

## 4.2 排版

1. 页面标题：`AppTheme.Typography.sheetTitle`
2. 卡片标题：`AppTheme.Typography.cardTitle`
3. 卡片正文：`AppTheme.Typography.cardBody`
4. 时间与提示：`AppTheme.Typography.meta`
5. 主按钮：`AppTheme.Typography.primaryButton`

## 4.3 圆角

1. 时间线标准卡片：`AppTheme.Radius.card`
2. 图片卡片：`AppTheme.Radius.image`
3. Sheet 内部按钮：20
4. 胶囊与轻提示：连续圆角，保持柔和

## 4.4 阴影

统一使用轻阴影：

1. `Color.black.opacity(0.05)`
2. radius 约 `10`
3. y 约 `5`

## 4.5 间距

1. 页面水平内边距：24
2. 首页水平内边距：20
3. 卡片间距：16
4. 区块纵向间距：24
5. 浮层与底部胶囊垂直关系以现有 Home 布局为准

# 6. HF-01 首页默认时间线态

## 6.1 设计目标

让用户感知到这仍然是一个安静的成长时间线，而非管理列表。

## 6.2 布局结构

从上到下：

1. 顶部模块导航
2. 情绪 Header
3. 时间线卡片列表
4. 底部悬浮操作胶囊

## 6.3 卡片视觉

标准记录卡维持当前语言：

1. 白色或深色卡面
2. 左侧轻量图标容器
3. 中部标题与副文案
4. 右侧淡化时间
5. 非照片卡不出现图片预览

卡片视觉应让用户第一眼看到：

1. 事件类型
2. 事件内容
3. 发生时间

## 6.4 禁止项

1. 禁止在卡片右侧常驻 `...`
2. 禁止常驻垃圾桶
3. 禁止在卡片底部露出“编辑”“删除”文字按钮

# 7. HF-02 首页 Cell 点按瞬时态

## 7.1 设计目标

通过极轻触觉与视觉反馈，让用户知道“这条记录已被选中，即将进入编辑”。

## 7.2 视觉反馈建议

建议在 `80ms - 120ms` 内完成以下任一组合：

1. 背景亮度轻微下压
2. 卡片整体缩放至 `0.985`
3. 阴影略微收紧

## 7.3 反馈边界

1. 不能出现弹跳
2. 不能出现明显位移
3. 不能出现水波纹式夸张反馈

---

# 8. HF-03 首页 Cell 长按菜单态

## 8.1 设计目标

用最低噪音呈现次级管理动作，让用户感受到“操作被允许”，但不让首页失去静谧感。

## 8.2 层级关系

1. 首页内容轻微退后
2. 当前被操作的 Cell 保持上下文存在感
3. 系统 context menu 作为焦点层出现

## 8.3 菜单视觉建议

采用原生 iOS context menu 风格，不另做大面板。

菜单项：

1. `编辑记录`
2. `删除记录`

## 8.4 菜单图标建议

1. 编辑：`square.and.pencil`
2. 删除：`trash`

## 8.5 视觉语气

1. 编辑项保持普通文本语气
2. 删除项使用系统 destructive 语义即可
3. 不额外加重视觉恐吓

# 9. HF-04 删除确认 Sheet

## 9.1 设计目标

在最后一步让用户确认“删的是哪条记录”，同时保持柔和、克制。

## 9.2 页面结构

从上到下：

1. 顶部留白与拖拽指示
2. 标题
3. 记录摘要卡
4. 说明文案
5. 主按钮 `删除`
6. 次按钮 `取消`

## 9.3 标题区

标题建议居中：

`删除这条记录？`

排版要求：

1. 使用 `sheetTitle`
2. 与摘要卡之间留出充足呼吸感
3. 不加副标题式长说明

## 9.4 记录摘要卡

摘要卡应复用首页卡片语义，但缩短为压缩版：

1. 主文案使用 `cardTitle`
2. 时间使用 `meta`
3. 卡面使用 `cardBackground`
4. 圆角建议 20 到 24

摘要卡不需要再完整还原图标与所有副文案，重点是帮助识别对象。

## 9.5 说明文案

建议使用两行以内的轻说明：

`删除后会从时间线移除。你仍可在短时间内撤销。`

视觉要求：

1. 使用 `secondaryText`
2. 居中或居左都可，但整体要简洁
3. 不使用危险说明图标

## 9.6 主按钮“删除”

按钮风格建议：

1. 保持与现有 `SheetPrimaryButton` 结构一致
2. 不改为鲜红实底按钮
3. 使用深色主文本底或克制的 destructive 文本按钮

若保持与当前主按钮体系一致，推荐：

1. 按钮底为 `primaryText`
2. 字色为白色
3. 风险通过文案与流程表达，而非颜色施压

## 9.7 次按钮“取消”

建议使用：

1. 文本按钮
2. 字色为 `secondaryText`
3. 与主按钮之间保留明显距离

# 10. HF-05 编辑成功反馈 Toast

## 10.1 设计目标

告诉用户修改已生效，但不打断当前浏览节奏。

## 10.2 呈现方式

1. 沿用首页轻反馈 Toast 样式
2. 位置在底部操作胶囊上方
3. 宽度与现有 Undo Toast 体系一致

## 10.3 文案

`已更新记录`

## 10.4 视觉结构

1. 卡片背景：`cardBackground`
2. 文案颜色：`primaryText`
3. 可不带图标
4. 停留时间短于 Undo Toast

## 10.5 动效

1. 自底部轻推入场
2. 轻淡出离场
3. 不需要用户点击

---

# 11. HF-06 删除成功 Undo Toast

## 11.1 设计目标

让用户明确删除已发生，并在极短时间内恢复该记录。

## 11.2 视觉结构

沿用现有 Undo Toast 语言：

1. 左侧为结果文案
2. 右侧为 `撤销`
3. 最右侧保留关闭按钮

## 11.3 文案

1. 主文案：`已删除记录`
2. 操作文案：`撤销`

## 11.4 样式建议

1. 背景使用 `cardBackground`
2. 文案使用 `primaryText`
3. 关闭图标使用 `tertiaryText`
4. 圆角与阴影与现有 Undo Toast 一致

## 11.5 停留时间

建议 `4s`

# 12. 编辑 Sheet 的统一视觉框架

四类编辑 Sheet 必须保持统一母版，避免每一类都像不同产品。

## 12.1 顶部结构

顶部统一为：

1. 左或中部标题 `编辑记录`
2. 右上关闭按钮
3. 必要时标题下方承接轻量分组控件

当前如果已有特殊 header 结构，应在编辑模式下尽量向统一母版收敛。

## 12.2 内容区规则

1. 首屏优先露出时间字段
2. 核心输入区域使用卡片化分组
3. 不做复杂边框表单
4. 不堆叠多余说明文本

## 12.3 底部主按钮

统一使用：

1. 文案：`保存修改`
2. 高度约 54 到 56
3. 圆角 20
4. 启用态为深色实底
5. 禁用态为 `primaryText.opacity(0.14)`

## 12.4 背景与留白

1. Sheet 背景使用 `background`
2. 水平留白 24
3. 垂直节奏遵循现有 `BaseRecordSheet`

# 13. 时间字段的高保真要求

时间是编辑页中的一级信息，应始终放在首屏上半区。

## 13.1 外观建议

时间行应呈现为轻卡片行：

1. 左侧标签：`时间`、`开始时间` 或 `结束时间`
2. 右侧当前值
3. 末端轻箭头或控件入口感

## 13.2 视觉语气

1. 不做厚重输入框
2. 不做系统表单列表感的强分隔线
3. 更像静态信息卡片上的可编辑行

## 13.3 交互反馈

点击时间行时：

1. 给予轻触感
2. 唤起原生日期时间选择器
3. 选择后就地更新显示文本

# 14. HF-07 奶量编辑 Sheet

## 14.1 设计目标

在编辑态下维持奶量记录的快速、无压力输入感，不让它变成厚重表单。

## 14.2 页面结构

从上到下：

1. 标题与关闭
2. 时间卡片行
3. 喂养方式切换
4. 母乳或瓶喂内容卡
5. 底部 `保存修改`

## 14.3 喂养方式切换

建议延续现有胶囊式切换感：

1. 容器背景轻淡
2. 当前选项使用更高对比度
3. 过渡平滑

## 14.4 母乳编辑区

视觉上建议像两块并列的小卡片：

1. 左侧时长
2. 右侧时长

每个区域都应简洁，不出现复杂计时仪表盘。

## 14.5 瓶喂编辑区

建议延续现有“中间数值 + 两侧步进”结构：

1. 数值居中
2. 两侧控制轻量
3. 不增加多余说明

# 15. HF-08 尿布编辑 Sheet

## 15.1 设计目标

让尿布编辑成为四类中最短、最安静的一种。

## 15.2 页面结构

从上到下：

1. 标题与关闭
2. 时间卡片行
3. 三个类型卡片按钮
4. 底部 `保存修改`

## 15.3 类型卡片视觉

每个选项建议为轻卡片：

1. 默认态为 `cardBackground`
2. 选中态通过轻底色、字重或边界强调
3. 不使用鲜艳色块区分三种类型

## 15.4 信息密度

1. 主标题足够
2. 辅助文案尽量短
3. 整页停留在一屏内完成

# 16. HF-09 睡眠编辑 Sheet

## 16.1 设计目标

突出“编辑真实时间”，而不是让用户直接改一个抽象时长数字。

## 16.2 页面结构

从上到下：

1. 标题与关闭
2. 开始时间卡片行
3. 结束时间卡片行
4. 时长结果卡
5. 底部 `保存修改`

## 16.3 时长结果卡

该卡片的作用是“确认结果”，不是输入框。

视觉建议：

1. 背景与普通卡一致
2. 文案更克制
3. 主数值可略大于正文

## 16.4 错误态

若结束时间早于开始时间：

1. 在相关时间行下方就地出现轻错误文案
2. 使用 `secondaryText` 或克制的暖色提醒
3. 不使用亮红块状警告

# 17. HF-10 辅食编辑 Sheet

## 17.1 设计目标

把标签、备注、照片整合为一张安静的饮食记录编辑页，避免母婴 App 的花哨感。

## 17.2 页面结构

从上到下：

1. 标题与关闭
2. 时间卡片行
3. 标签区
4. 备注区
5. 照片区
6. 底部 `保存修改`

## 17.3 标签区

标签建议使用低对比度 chip：

1. 默认态为浅底或描边感
2. 选中态通过轻度底色加强
3. 不同标签之间不使用彩色区分

## 17.4 备注区

建议采用一张柔和的文本卡：

1. 背景与页面有轻微层级差
2. 文本颜色为 `primaryText`
3. 占位文案为 `secondaryText`

## 17.5 照片区

建议表现为一张安静的大图卡：

1. 有图时优先展示图片
2. 无图时展示浅层级占位
3. `删除图片` 与 `重选图片` 为次级文本按钮

# 18. 深色模式要求

本稿所有界面必须完整适配深色模式。

## 18.1 基本原则

1. 不直接反相
2. 继续保持温暖、柔和、低刺激
3. 对比度要足够，但不能变成纯黑纯白

## 18.2 具体要求

1. 背景使用深暖灰而非纯黑
2. 卡片使用更浅一级深灰
3. 主文案使用暖白
4. 辅助文案通过透明度降级

## 18.3 删除相关界面

即使在深色模式下，删除确认与删除完成反馈也不应切换到亮红高对比方案。

# 19. 动效与触感说明

## 19.1 总原则

动效只是帮助用户理解状态切换，不承担戏剧化表达。

## 19.2 点按进入编辑

1. 轻触感反馈
2. 卡片轻压
3. Sheet 使用系统上拉转场

## 19.3 长按菜单

1. selection 级别触感
2. 使用系统 context menu 出场

## 19.4 删除确认

1. 底部 Sheet 平滑上浮
2. 不加入危险抖动

## 19.5 删除完成

1. Cell 淡出并收缩
2. 邻近卡片自然上移
3. Undo Toast 自底部轻推入

## 19.6 撤销恢复

1. 恢复卡片回插原位
2. 使用轻 spring
3. 不做庆祝式成功反馈

# 21. 设计稿验收清单

高保真设计稿至少要满足以下条件：

1. 首页默认态无常驻管理按钮
2. 点按态与长按态的反馈强度足够轻
3. 长按菜单只有两项
4. 删除确认为底部 Sheet，而非 Alert
5. 删除确认页中能清晰识别被删除对象
6. 编辑成功和删除成功反馈视觉语言统一
7. 四类编辑 Sheet 有统一母版
8. 所有类型的时间字段均位于首屏上半区
9. 深色模式可读且不刺眼
10. 删除相关页面没有亮红警报感

---

# 22. 开发映射建议

为减少设计与开发偏差，建议在实现时按以下组件映射：

1. 首页记录卡继续复用现有 `StandardRecordCard` 与 `FoodPhotoCard`
2. 点按反馈作为卡片态扩展，不新增视觉组件
